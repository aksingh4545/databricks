Databricks Problem Solving Guide

This guide is designed for engineers who get confused while working in Databricks. Whenever you are stuck, follow this document step-by-step.

1. Mental Model (Most Important)

Databricks = Compute Engine Data Lives In: Cloud Storage | +-- Parquet files +-- Delta Tables Unity Catalog | +-- Catalog | +-- Schema | +-- Tables

Remember this rule:

Databricks does NOT store your data. It only processes the data stored in cloud storage.

2. Quick Start Workflow (Follow This Always)

If you want to build a table in Databricks follow this order.


Step 1 → Create Catalog
Step 2 → Create Schema
Step 3 → Create Delta Table
Step 4 → Insert Data
Step 5 → Query Data

3. Step 1 — Create Catalog


CREATE CATALOG company_data;
Check catalogs: 

SHOW CATALOGS;
Use catalog: 

USE CATALOG company_data;

4. Step 2 — Create Schema


CREATE SCHEMA sales;
Check schemas: 

SHOW SCHEMAS;
Use schema: 

USE SCHEMA sales;

5. Step 3 — Create Delta Table


CREATE TABLE orders (
order_id INT,
product STRING,
price INT
)
USING DELTA;
Query table: 

SELECT * FROM orders;

6. What USING DELTA Actually Does

orders/ part-0001.parquet part-0002.parquet _delta_log/ 000000.json 000001.json

Parquet files store the data. _delta_log tracks changes.

7. Managed vs External Table

Managed Table


CREATE TABLE orders(
id INT,
product STRING
)
USING DELTA;
Behavior:
  • Databricks controls storage
  • DROP TABLE deletes data

External Table


CREATE TABLE orders
USING DELTA
LOCATION '/mnt/orders';
Behavior:
  • You control storage location
  • DROP TABLE keeps data

8. How to See Where Data Is Stored

Run this: 

DESCRIBE DETAIL orders;
This shows:
  • table format
  • storage location
  • table size

9. Debugging Checklist

If something fails follow this checklist.
Problem: Table not found
Check catalog: SHOW CATALOGS Check schema: SHOW SCHEMAS Check tables: SHOW TABLES
Problem: Data not showing
Check table history DESCRIBE HISTORY orders
Problem: Wrong table location
Check storage path DESCRIBE DETAIL orders

10. Important Delta Commands

Insert Data 

INSERT INTO orders VALUES
(1,'Laptop',50000),
(2,'Phone',20000);
Update 

UPDATE orders
SET price = 60000
WHERE product='Laptop';
Delete 

DELETE FROM orders
WHERE order_id=2;

11. Delta Time Travel

View table history: 

DESCRIBE HISTORY orders;
Query old version: 

SELECT * FROM orders VERSION AS OF 0;

12. Medallion Architecture

Raw Data | v Bronze | v Silver | v Gold

Bronze → raw data Silver → cleaned data Gold → business analytics

13. Full End-to-End Example


CREATE CATALOG demo;

CREATE SCHEMA demo.sales;

CREATE TABLE demo.sales.orders
(
id INT,
product STRING,
price INT
)
USING DELTA;

INSERT INTO demo.sales.orders
VALUES (1,'Laptop',50000);

SELECT * FROM demo.sales.orders;
